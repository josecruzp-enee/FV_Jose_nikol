========================================================
PANELES FV — OVERVIEW DEL MÓDULO
Ubicación: electrical/paneles/
========================================================

PROPÓSITO
--------------------------------------------------------
Este módulo implementa el dominio eléctrico del generador
fotovoltaico (lado DC).

Convierte:

    especificación de panel + inversor + condiciones

en:

    una configuración eléctrica válida del sistema FV

Es la fuente de verdad eléctrica base del sistema.


PRINCIPIO ARQUITECTÓNICO
--------------------------------------------------------
EntradaPaneles  = problema eléctrico
ResultadoPaneles = solución eléctrica

El dominio es:

    determinista
    tipado
    sin efectos colaterales

No usa dict.
No mezcla responsabilidades.
No contiene lógica NEC.


QUÉ RESUELVE
--------------------------------------------------------
Panel + inversor + temperatura + configuración
                          ↓
Generador FV definido:

- número de módulos en serie (Ns)
- número de strings (Np)
- voltajes (Vmp, Voc frío)
- corrientes base (Imp, Isc)
- distribución por MPPT


FLUJO INTERNO
--------------------------------------------------------
EntradaPaneles
        ↓
validacion_strings
        ↓
dimensionado_paneles
        ↓
calculo_de_strings
        ↓
orquestador_paneles
        ↓
ResultadoPaneles


QUÉ CALCULA EL DOMINIO
--------------------------------------------------------
- Voc en condiciones frías
- Vmp en condiciones de operación
- límites eléctricos por inversor (MPPT y Vdc_max)
- número óptimo de módulos en serie
- número total de strings
- paneles no utilizados (warnings)
- topología base del sistema


QUÉ NO HACE (FUERA DE ALCANCE)
--------------------------------------------------------
Este dominio NO:

- aplica factores NEC (125%, etc.)
- dimensiona conductores
- selecciona protecciones
- calcula caída de voltaje
- dimensiona canalización
- calcula energía (kWh)
- selecciona inversores

Solo define el generador FV DC.


ARCHIVOS PRINCIPALES
--------------------------------------------------------
entrada_panel.py
    Contrato de entrada del dominio (tipado).

validacion_strings.py
    Validaciones eléctricas y de coherencia.

dimensionado_paneles.py
    Define cantidad de paneles y potencia DC.

calculo_de_strings.py
    Motor eléctrico principal (strings).

orquestador_paneles.py
    Orquestador del dominio.
    Coordina todo el flujo.

resultado_paneles.py
    Contrato de salida (fuente de verdad).


ENTRYPOINT OFICIAL
--------------------------------------------------------
Usar siempre:

    ejecutar_paneles()

NO llamar directamente:

    calculo_de_strings
    dimensionado_paneles


SALIDA ESTÁNDAR
--------------------------------------------------------
ResultadoPaneles (inmutable)

Contiene:

    ok
    errores[]
    warnings[]

    array:
        potencia_dc_w
        vdc_nom
        idc_nom
        isc_total
        voc_frio_array_v
        n_strings_total
        n_paneles_total

    strings[]:
        detalle por string

    recomendacion:
        configuración óptima

    meta:
        resumen del sistema


IDEA CLAVE DEL DISEÑO
--------------------------------------------------------
Paneles = modelo físico del generador FV.

Define:

    voltajes
    corrientes base
    topología

Todo lo demás depende de este resultado.


REGLAS DEL DOMINIO
--------------------------------------------------------
- NO usar dict
- SOLO dataclass tipado
- Resultado inmutable (frozen=True)
- NO lógica NEC
- NO dependencias externas a paneles/modelos
- SOLO un punto de salida (ResultadoPaneles)


INTEGRACIÓN CON EL SISTEMA
--------------------------------------------------------
ResultadoPaneles alimenta:

    → conductores
    → protecciones
    → análisis posteriores

Nunca se recalcula fuera del dominio.


ESTADO DEL MÓDULO
--------------------------------------------------------
ESTABLE — DOMINIO CERRADO

Permitido:
- corrección de bugs
- mejoras internas del motor

No permitido:
- cambiar contratos (entrada/salida)
- introducir NEC
- romper flujo Entrada → Resultado
